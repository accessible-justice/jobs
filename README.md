## Software Engineer (Full-Stack / Backend)

**Remote · TypeScript + Node.js**

---

### 关于我们 / About Us

Accessible Justice 致力于消除人们获得法律帮助的壁垒。
我们相信技术可以让司法变得更及时、更公平——无论用户的背景或资源如何。

Accessible Justice is removing barriers between people and legal help.
We believe technology can make justice more timely and more equitable — regardless of a user's background or resources.

---

### 这个角色 / The Role

我们正在寻找一位能快速交付、注重产品质量的工程师。
你将深度参与产品功能的设计与落地，和团队一起快速迭代，直接影响真实用户的体验。

We're looking for an engineer who ships fast and cares about quality.
You'll be hands-on in feature design and delivery, iterating quickly alongside the team and directly shaping the experience of real users.

---

### 职责 / Responsibilities

**中文**

- 快速、可靠地交付面向用户的功能
- 设计并实现清晰、稳健的 REST API 接口
- 在快速变化的产品节奏下保持代码结构清晰
- 主动发现并解决用户体验或系统层面的问题
- 在 PR 和异步沟通中清楚表达技术决策
- 与产品、设计紧密协作，理解为什么而非仅仅是什么

**English**

- Ship user-facing features quickly and reliably
- Design and implement clean, robust REST API endpoints
- Keep code well-structured in a fast-moving product environment
- Proactively identify and fix issues — in UX and in systems
- Communicate technical decisions clearly in PRs and async discussions
- Work closely with product and design — understand the *why*, not just the *what*

---

### 我们在找的人 / What We're Looking For

**中文**

- 3 年以上 TypeScript / Node.js 工程经验
- 能在快速迭代的节奏中稳定交付
- 重视公平性：在技术决策中考虑可及性与包容性
- 代码整洁，有分层架构意识
- 书面沟通清晰，能用文字推动异步协作
- 对产品有判断力——不只执行，还会提问

**English**

- 3+ years of hands-on experience with TypeScript / Node.js
- Delivers consistently in a fast, iterative product environment
- Values equity: considers accessibility and inclusivity in technical decisions
- Writes clean, layered code that others can maintain
- Strong written communication — moves work forward asynchronously
- Has product instincts — asks *why* before implementing *what*

我们通过一个小型工程练习了解你的工作方式——不是算法题，是真实的工程问题，大约 2–4 小时。

We evaluate candidates through a short engineering exercise — not algorithm puzzles, but real product engineering problems. Expect 2–4 hours.

Complete Issue 1, then choose from the rest. Submit via pull request — use the PR description to walk us through your thinking.
